# Security policy

## Supported versions

Security updates are applied to the **`main`** branch and released according to the project’s normal versioning process ([Commitizen](https://commitizen-tools.github.io/commitizen/) / [CHANGELOG.md](CHANGELOG.md)).

## Reporting a vulnerability

**Do not** open a public GitHub issue for security vulnerabilities.

Please report them through one of these channels:

1. **[GitHub Security Advisories](https://docs.github.com/en/code-security/security-advisories)** — use **“Report a vulnerability”** on this repository if the feature is enabled.
2. **Maintainers** — contact the maintainers privately (see repository profile or org contacts).

Include enough detail to reproduce or assess the issue (affected component, version or commit, and impact). We will acknowledge receipt and coordinate a fix and disclosure timeline.

## General expectations

- Follow responsible disclosure: allow time for a fix before public details.
- See [CONTRIBUTING.md](CONTRIBUTING.md) for the code of conduct and how we handle contributions.
